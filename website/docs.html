Whois Hacker 使用文档

安装指南

有多种方式可以安装和使用 Whois Hacker：

作为 Go 库使用

如果您是 Go 开发者，可以直接在您的项目中引入 Whois Hacker 包：

go get github.com/cyberspacesec/whois-hacker

然后在代码中使用：

package main

import (
    "fmt"
    "github.com/cyberspacesec/whois-hacker/pkg/whois"
)

func main() {
    // 单个域名查询
    result, err := whois.ExecuteQuery(&whois.QueryOptions{
        Domain: "example.com",
        ValidateResult: true,
    })
    if err != nil {
        fmt.Printf("查询失败: %v\n", err)
        return
    }
    fmt.Printf("域名: %s\n", result.Domain.Domain)
    fmt.Printf("注册商: %s\n", result.Registrar.Name)
    fmt.Printf("创建时间: %s\n", result.Domain.CreatedDate)
    fmt.Printf("过期时间: %s\n", result.Domain.ExpirationDate)

    // 使用代理查询
    result, err = whois.ExecuteQuery(&whois.QueryOptions{
        Domain: "example.com",
        UseProxy: true,
        ValidateResult: true,
    })

    // 批量查询
    aggregator := &whois.QueryAggregator{
        concurrency: 5,
    }
    domains := []string{"example.com", "google.com", "github.com"}
    for _, domain := range domains {
        aggregator.AddQuery(domain, &whois.QueryOptions{
            ValidateResult: true,
            Priority: 1,
        })
    }
    results := aggregator.ExecuteAll()

    // 使用缓存
    cache := whois.GetCache()
    if entry, exists := cache.Get("example.com"); exists {
        fmt.Printf("从缓存获取: %+v\n", entry)
    }

    // 设置代理
    proxyConfig := &whois.ProxyConfig{
        Enabled: true,
        Type: "socks5",
        Address: "127.0.0.1:1080",
        Username: "user",
        Password: "pass",
        Timeout: 10,
    }
    if err := whois.SetWhoisProxy(proxyConfig); err != nil {
        fmt.Printf("设置代理失败: %v\n", err)
    }

    // 查询 ASN 信息
    ipv4Ranges, ipv6Ranges, err := whois.GetIPRangesByASN("AS15169")
    if err != nil {
        fmt.Printf("查询ASN失败: %v\n", err)
        return
    }
    fmt.Printf("IPv4范围: %v\n", ipv4Ranges)
    fmt.Printf("IPv6范围: %v\n", ipv6Ranges)

通过 Go 安装命令行工具

                    
如果您已经安装了 Go 环境，可以通过以下命令直接安装：

                    
go install github.com/cyberspacesec/whois-hacker/cmd/whois-hacker@latest

基本使用

Whois Hacker 提供了多种命令和模式，以下是基本的命令结构：

whois-hacker [command] [flags]

主要命令包括：

  • query：查询单个域名
  • batch：批量查询多个域名
  • service：启动 API 服务
  • version：显示版本信息

全局标志：

  • --debug：启用调试模式
  • --help：显示帮助信息

域名查询

查询单个域名的 WHOIS 信息：

whois-hacker query example.com

使用指定的 WHOIS 服务器：

whois-hacker query example.com --server whois.verisign-grs.com

使用不同的输出格式：

whois-hacker query example.com --output-format json

支持的输出格式包括：

  • text：原始文本
  • json：JSON 格式
  • csv：CSV 格式

批量查询

批量查询多个域名：

whois-hacker batch -f domains.txt --output-format json --output-file results.json

domains.txt 文件中每行一个域名，示例：

example.com
google.com
microsoft.com

控制并发查询数量：

whois-hacker batch -f domains.txt --concurrency 5

使用不同的输出目标：

whois-hacker batch -f domains.txt --output-file results.csv --output-format csv

输出格式

Whois Hacker 支持多种输出格式，可以使用 --output-format 标志指定：

文本格式 (text)

原始 WHOIS 服务器响应，包含所有信息：

whois-hacker query example.com --output-format text

JSON 格式 (json)

将 WHOIS 信息格式化为 JSON 对象，易于程序处理：

whois-hacker query example.com --output-format json

CSV 格式 (csv)

将结果输出为 CSV 格式，适合导入电子表格软件：

whois-hacker query example.com --output-format csv

输出到文件

使用 --output-file 将结果保存到文件：

whois-hacker query example.com --output-format json --output-file example.json

代理支持

Whois Hacker 支持通过代理服务器发送 WHOIS 查询：

HTTP 代理

whois-hacker query example.com --proxy http://proxy.example.com:8080

SOCKS5 代理

whois-hacker query example.com --proxy socks5://127.0.0.1:1080

带认证的代理

whois-hacker query example.com --proxy http://username:password@proxy.example.com:8080

代理配置也可以通过环境变量设置：

export WHOIS_PROXY="http://proxy.example.com:8080"
whois-hacker query example.com

国际化域名

Whois Hacker 原生支持国际化域名 (IDN)，可以直接查询包含非 ASCII 字符的域名：

whois-hacker query 例子.中国

程序会自动将国际化域名转换为 Punycode 格式进行查询。

API 服务

启动 RESTful API 服务：

whois-hacker service --port 8080

配置 API 速率限制：

whois-hacker service --rate-limit 10 --rate-limit-period 60

这将限制每个 IP 地址在 60 秒内最多发起 10 次请求。

API 服务支持以下端点：

  • GET /health：健康检查
  • GET /version：版本信息
  • GET /whois?domain=example.com：查询单个域名

使用 curl 测试 API：

curl "http://localhost:8080/whois?domain=example.com"

Docker 部署

使用 Docker 部署 Whois Hacker 服务：

使用官方镜像

docker run -p 8080:8080 cyberspacesec/whois-hacker service

使用 Docker Compose

创建 docker-compose.yml 文件：

version: '3'

services:
  whois-hacker:
    image: cyberspacesec/whois-hacker
    ports:
      - "8080:8080"
    command: service --port 8080 --rate-limit 100 --rate-limit-period 60
    restart: unless-stopped

然后启动服务：

docker-compose up -d

MCP 集成

Whois Hacker 支持 Multi-agent Conversation Protocol (MCP)，使其能够作为 AI 助手的工具使用。

MCP 端点

启动带有 MCP 支持的服务：

whois-hacker service --enable-mcp

MCP 服务提供以下端点：

  • POST /mcp/request_planning：创建请求并计划任务
  • POST /mcp/get_next_task：获取下一个待处理任务
  • POST /mcp/mark_task_done：标记任务为已完成
  • POST /mcp/approve_task_completion：批准任务完成
  • POST /mcp/approve_request_completion：批准请求完成

集成示例

AI 助手可以通过 MCP 协议实现自动化 WHOIS 查询，例如：

// 创建查询请求
POST /mcp/request_planning
{
  "originalRequest": "查询域名 example.com 的 WHOIS 信息",
  "tasks": [
    {
      "title": "查询 example.com",
      "description": "获取 example.com 的 WHOIS 信息"
    }
  ]
}

高级用法

自定义 WHOIS 服务器

为特定 TLD 指定 WHOIS 服务器：

whois-hacker query example.com --server whois.verisign-grs.com

超时设置

设置查询超时时间：

whois-hacker query example.com --timeout 30

重试机制

配置查询失败重试次数：

whois-hacker query example.com --retry 3

调试模式

启用调试模式查看详细日志：

whois-hacker query example.com --debug

常见问题

这可能有几个原因：

  • 域名注册商启用了隐私保护
  • WHOIS 服务器限制了查询频率
  • 注册商使用非标准格式提供 WHOIS 信息

您可以尝试使用 --server 标志直接指定该 TLD 的 WHOIS 服务器，或者启用 --debug 模式查看详细日志。

许多 WHOIS 服务器会限制查询频率。解决方法包括：

  • 使用批量查询时降低并发数 --concurrency 1
  • 使用代理服务器轮换 IP 地址
  • 在查询之间添加延迟 --query-delay 2

批量查询速度受多种因素影响：

  • WHOIS 服务器响应时间
  • 并发数设置（过高可能导致被限制）
  • 网络连接质量
  • 代理服务器性能（如使用代理）

建议尝试调整 --concurrency 参数找到最佳平衡点。
