Multi-agent Conversation Protocol (MCP) 集成

Whois Hacker 支持 Multi-agent Conversation Protocol (MCP)，使其能够作为 AI 助手和大型语言模型的工具使用。借助 MCP 集成，AI 系统可以直接与 Whois Hacker 通信，执行一系列任务，并获取 WHOIS 查询结果。

什么是 MCP？

Multi-agent Conversation Protocol (MCP) 是一种专为 AI 助手设计的协议，允许 AI 代理与外部工具和服务进行结构化通信。通过 MCP，AI 助手可以：

  • 创建并管理复杂的任务序列
  • 执行特定的外部功能和服务
  • 获取任务执行的反馈和结果
  • 按照特定的工作流程组织和管理任务

Whois Hacker 实现了 MCP 协议，使 AI 系统能够直接与其交互，执行 WHOIS 查询任务，并获取结构化结果。

启用 MCP 支持

要在 Whois Hacker 中启用 MCP 支持，您需要在启动服务时添加 --enable-mcp 标志： 

whois-hacker service --port 8080 --enable-mcp

MCP 功能默认在 API 服务的 /mcp 路径下提供。您可以使用 --mcp-path 标志自定义路径： 

whois-hacker service --port 8080 --enable-mcp --mcp-path /ai-tools

这将使 MCP 端点可在 http://localhost:8080/ai-tools/ 下访问。

MCP 工作流程

MCP 的工作流程遵循一个结构化的步骤序列，从请求计划开始，到任务执行和结果获取结束。以下是在 Whois Hacker 中使用 MCP 的典型流程：

1
请求计划: AI 助手创建一个请求，并定义要执行的任务列表。例如，计划查询多个域名的 WHOIS 信息。
2
获取任务: AI 助手获取下一个待执行的任务。Whois Hacker 系统会返回任务详情和执行指南。
3
执行任务: AI 助手执行任务，例如发起 WHOIS 查询。查询结果将由 Whois Hacker 处理和存储。
4
标记完成: 任务完成后，AI 助手标记任务为已完成，并可以包含详细的完成信息。
5
批准完成: 用户批准任务完成，确认结果符合预期。
6
循环执行: AI 助手获取下一个任务，重复步骤 2-5，直到所有任务完成。
7
完成请求: 所有任务完成后，整个请求被标记为完成，并汇总所有结果。

这种工作流程确保了 AI 助手与 Whois Hacker 之间的交互是结构化和可预测的，使 AI 系统能够有效地执行复杂的 WHOIS 查询任务。

MCP 端点

Whois Hacker 实现了以下 MCP 端点，用于支持完整的 MCP 工作流程：

POST /mcp/request_planning

创建新的请求并规划相关任务。您必须提供 'originalRequest' 和 'tasks'，可选择提供 'splitDetails'。

请求体

{
  "originalRequest": "查询多个域名的 WHOIS 信息",
  "tasks": [
    {
      "title": "查询 example.com",
      "description": "获取 example.com 的 WHOIS 信息"
    },
    {
      "title": "查询 google.com",
      "description": "获取 google.com 的 WHOIS 信息"
    }
  ],
  "splitDetails": "将查询拆分为多个任务"
}

响应

{
  "requestId": "req-abc123",
  "tasks": [
    {
      "taskId": "task-1",
      "title": "查询 example.com",
      "description": "获取 example.com 的 WHOIS 信息",
      "status": "pending"
    },
    {
      "taskId": "task-2",
      "title": "查询 google.com",
      "description": "获取 google.com 的 WHOIS 信息",
      "status": "pending"
    }
  ],
  "progress": {
    "total": 2,
    "completed": 0,
    "pending": 2
  }
}

POST /mcp/get_next_task

获取下一个待处理的任务。如果所有任务都已完成，将指示无更多任务并等待请求完成批准。

请求体

{
  "requestId": "req-abc123"
}

响应

{
  "task": {
    "taskId": "task-1",
    "title": "查询 example.com",
    "description": "获取 example.com 的 WHOIS 信息",
    "status": "pending"
  },
  "progress": {
    "total": 2,
    "completed": 0,
    "pending": 2
  }
}

POST /mcp/mark_task_done

将任务标记为已完成。您可以可选地提供完成详情。

请求体

{
  "requestId": "req-abc123",
  "taskId": "task-1",
  "completedDetails": "已成功查询 example.com 的 WHOIS 信息，域名注册于 1995 年，属于 IANA。"
}

响应

{
  "task": {
    "taskId": "task-1",
    "title": "查询 example.com",
    "description": "获取 example.com 的 WHOIS 信息",
    "status": "done",
    "completedDetails": "已成功查询 example.com 的 WHOIS 信息，域名注册于 1995 年，属于 IANA。"
  },
  "progress": {
    "total": 2,
    "completed": 1,
    "pending": 1
  }
}

POST /mcp/approve_task_completion

批准任务完成。用户必须调用此工具来确认任务确实已完成。

请求体

{
  "requestId": "req-abc123",
  "taskId": "task-1"
}

响应

{
  "task": {
    "taskId": "task-1",
    "title": "查询 example.com",
    "description": "获取 example.com 的 WHOIS 信息",
    "status": "approved",
    "completedDetails": "已成功查询 example.com 的 WHOIS 信息，域名注册于 1995 年，属于 IANA。"
  },
  "progress": {
    "total": 2,
    "completed": 1,
    "pending": 1
  }
}

POST /mcp/approve_request_completion

}

与 Claude 集成

通过 Claude 的工具使用功能，您可以将 Whois Hacker MCP 作为外部工具连接到 Claude： 

{

    {
      "name": "mcp__request_planning",
      "description": "创建新的 WHOIS 查询请求并规划任务",
      "parameters": {
        "type": "object",
        "properties": {
          "originalRequest": { "type": "string" },
          "tasks": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "title": { "type": "string" },
                "description": { "type": "string" }
              }
            }
          }
        },
        "required": ["originalRequest", "tasks"]
      }
    },
    // 其他 MCP 工具定义...
  ]
}

与 GPT 集成

使用 OpenAI 的 Function Calling 功能，您可以在 GPT 模型中集成 Whois Hacker： 

{
  "functions": [
    {
      "name": "query_whois",
      "description": "查询域名的 WHOIS 信息",
      "parameters": {
        "type": "object",
        "properties": {
          "domain": {
            "type": "string",
            "description": "要查询的域名"
          }
        },
        "required": ["domain"]
      }
    }
  ]
}

函数实现将使用 MCP API 来执行查询并获取结果。

自动化脚本

您可以创建 Python 脚本来自动执行 MCP 工作流程： 

import requests

MCP_BASE_URL = "http://localhost:8080/mcp"

# 创建请求
response = requests.post(f"{MCP_BASE_URL}/request_planning", json={
    "originalRequest": "监控竞争对手域名",
    "tasks": [
        {"title": "查询 competitor1.com", "description": "..."},
        {"title": "查询 competitor2.com", "description": "..."}
    ]
})
request_id = response.json()["requestId"]

# 获取并执行任务
while True:
    next_task = requests.post(f"{MCP_BASE_URL}/get_next_task", 
                             json={"requestId": request_id}).json()
    
    if next_task.get("all_tasks_done"):
        break
        
    task = next_task["task"]
    # 执行 WHOIS 查询
    domain = task["title"].split(" ")[1]
    whois_result = query_whois(domain)
    
    # 标记任务完成
    requests.post(f"{MCP_BASE_URL}/mark_task_done", json={
        "requestId": request_id,
        "taskId": task["taskId"],
        "completedDetails": f"查询结果: {whois_result}"
    })
    
    # 申请批准
    requests.post(f"{MCP_BASE_URL}/approve_task_completion", 
                 json={"requestId": request_id, "taskId": task["taskId"]})

# 完成整个请求
requests.post(f"{MCP_BASE_URL}/approve_request_completion", 
             json={"requestId": request_id})

与企业应用集成

您可以将 Whois Hacker MCP 与企业应用集成，例如网络安全平台或资产管理系统：

  1. 创建工作流，定期查询企业域名资产
  2. 使用 MCP API 自动执行查询
  3. 将结果导入到安全监控系统
  4. 根据到期日期创建提醒和通知
  5. 生成安全报告和合规文档

这使您的企业系统能够自动跟踪和监控域名资产，提前发现安全风险和到期问题。

最佳实践

任务设计
  • 为每个域名创建单独的任务
  • 使用清晰的任务标题和描述
  • 在描述中包含查询目的和要提取的信息
  • 对大量域名使用批处理任务
请求管理
  • 定期检查请求状态
  • 为长时间运行的请求设置超时机制
  • 使用 list_requests 监控活动请求
  • 在完成所有任务后批准请求
错误处理
  • 实施重试机制处理临时故障
  • 记录查询错误并在完成详情中提供
  • 为常见错误（如限速、无数据）创建恢复策略
  • 监控批量查询的成功率
性能优化
  • 对大量域名使用合理的并发设置
  • 在连续请求之间添加延迟避免限速
  • 缓存最近查询结果减少重复请求
  • 对相似任务分组批处理
